🐝 BeePR

Python Discord.py Bot Work in Progress

A multi-purpose Discord bot packed with entertainment and utility features. BeePR combines a powerful music system, fun commands, and useful tools all in one package. It's actively being developed with new features being added regularly.

⚙️ Status: Currently under heavy development. Features, commands, and the command structure are subject to change. The source code is not publicly available at this time.

✨ Features

🎵 Music System

  • Play songs or playlists from YouTube
  • Queue management with shuffle and loop controls
  • Volume control and playback speed adjustment
  • Bass boost and audio effects (Nightcore, Slowed)

🖼️ Image Commands

  • Fetch random SFW images from e926.net with custom tags
  • Fetch NSFW images from e621.net (NSFW channels only)

🛠️ Utility & Automation

  • Bot latency checking and diagnostics
  • Server information and statistics
  • User profile and member details
  • Automatic x.com link conversion to fixupx.com for better embeds

📋 Command Reference

Prefix: b! or B!

Music Commands

b!join Joins your voice channel
b!play <url or search> Plays a song or adds playlist to queue
b!pause Pauses current playback
b!resume Resumes paused song
b!skip Skips to next song
b!queue Shows the current queue
b!shuffle Randomizes queue order
b!loop Toggles loop mode
b!stop Stops music and clears queue
b!bassboost <level> Applies bass boost (off, on, ultra)
b!speed <level> Changes speed (normal, slowed, nightcore)
b!leave Leaves voice channel

Fun Commands

b!e9pic [tags] Fetch random SFW image from e926.net
b!e6pic [tags] Fetch random NSFW image from e621.net

Utility Commands

b!ping Shows bot latency
b!serverinfo Displays server information
b!userinfo [member] Shows user or member details
b!help [command] Shows help for commands

📗 Tech Stack

  • Python 3.8+ - Core language
  • discord.py - Discord API wrapper
  • yt-dlp - YouTube audio extraction
  • aiohttp - Async HTTP requests
⚠️ Source Code: Currently not open-source. This is partly due to library licensing concerns and the heavy WIP nature of the project. Check back later for potential updates to this status.
